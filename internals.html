SpareTime

This application was built using the Django framework using the MVC (or Model-View-Template in Django) architecture. The entire applicationn is wrapped in a virtual environment (venv) so that the installed plug-ins are contained within the application. The requirements for this project can be found in requirements.txt and can be installed via pip. Django itself can also be installed with pip.

Our entire codebase can be find within the mysite folder. We will go folder-by-folder and file by file to describe how the system is built.

  • folder: database This folder holds our sqlite database (called data) that is used in during our local development.
  • file: manage.py This file is a django-generated file that holds the code for various django-python functions such as how to sync the database (python manage.py syncdb) and how to run the localhost server (python manage.py runserver).
  • folder: mysite
    • file: __init__.py
      This file is a django-generated file that was unaltered.
    • file: settings.py
      This file is the main file that holds the django settings for the sparetime application. These settings include the database information, directory for static files, directory for templates, installed django apps, and heroku settings at the bottom.
    • file: urls.py
      This file is the top level url re-routing file. Any url essentially re-routes to the url file in the socialcalendar app
    • file: wsgi.py
      This is a django-generated file that was unaltered.
  • folder: socialcalendar
    • file: __init__.py
      This file is a django-generated file that was unaltered.
    • file: models.py
      This is the the main model file. It holds the information for our five database tables: ExceptionDate, Event, UserProfile, Name, and Comment.
    • file: tests.py
      This file is a django-generated file that was unaltered.
    • file: urls.py
      This is the url re-routing file for socialcalendar.
    • file: views.py
      This is the main view file for our application. It contains all of the django-python methods that are the base of the functionality
  • folder: static
    • folder: bootstrap
      A downloaded copy of all bootstrap templating files.
    • folder: css
      • file: custom.css
        The main css file for the application
      • file: homepage.css
        The css for the homepage (not logged-in)
      • file: tinyscrollbar.css
        The css for the scrollbar used in our calendar
    • folder: img
      Contains all of the images used in our application.
    • folder: js
      • file: draggable.js
        Contains most of the javascript for our application.
      • file: facebook.js
        Contains the facebook javascript
      • file: jquery.tinyscrollbar.min.js
        Contains the jquery code for the scrollbar in our calendar
    • folder: timePicker
      Contains the css and js files for the default date and time picker.
  • folder: templates
    • file: homepage.html
      This is the html file for our default (not logged-in) homepage
    • folder: socialcalendar
      • file: calendar.html
        The html file for our calendar
      • file: event.html
        The html file for individual events
      • file: monthly.html
        The html file for the monthly view on the calendar
      • file: weekly.html
        The html file for the weekly view on the calendar
    • file: template.html
      This is the html file that serves as the main template for our application. It contains the code for the header and footer that is the same on every page of our application.
